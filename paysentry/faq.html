PaySentry FAQ

Everything you need to know about controlling AI agent spending, preventing prompt injection, and enforcing payment policies.

What is PaySentry?

PaySentry is an open-source control plane that sits between AI agents and payment protocols (x402, ACP, AP2, Visa TAP). It provides real-time spending monitoring, declarative policy enforcement, and automated dispute resolution for autonomous agent transactions.

Think of it as the "firewall for agent wallets" — PaySentry observes what agents spend, controls how they spend it, and protects when things go wrong.

How do I add spending limits to x402 payments?

Use PaySentry's PolicyEngine with RuleBuilder to define deterministic spending limits: 

import { RuleBuilder } from '@paysentry/control';

const policy = RuleBuilder.create('daily-budget')
  .maxAmount(100)        // Max $100 per transaction
  .dailyBudget(1000)     // Max $1000 per day
  .currencies('USDC')
  .action('allow')
  .build();

paysentry.addPolicy(policy);

These limits are enforced in code, not prompts — so even if an AI agent is compromised via prompt injection, it cannot override the spending policy.

How does PaySentry prevent prompt injection attacks on agent wallets?

PaySentry enforces deterministic policies written in code, not prompts. Even if an AI agent's context is compromised via prompt injection (e.g., a hidden instruction on a webpage tells the agent to make a $499 payment), the PolicyEngine evaluates the transaction before funds move and blocks it if it violates:

  • Budget caps (daily/weekly/monthly limits)
  • Per-transaction maximums
  • Recipient allowlists/denylists
  • Approval requirements for high-value transactions

Result: The attack is contained to a single blocked transaction, not a drained wallet.

What's the difference between PaySentry and protocol-native wallets?

Protocol-native wallets (like x402 integrations) handle transaction execution. PaySentry handles transaction governance:

  • Observability — What is my agent spending? Real-time dashboards, alerts, cost attribution
  • Control — Declarative policies no LLM can override (budget caps, allowlists, approval chains)
  • Protection — Audit trails for forensics, automated dispute resolution

PaySentry sits between your agent framework and the wallet, acting as a protocol-agnostic control plane. It works with x402, ACP, AP2, Visa TAP, etc.

Does PaySentry work with all agent payment protocols?

Yes. PaySentry is protocol-agnostic and supports:

  • x402 — Coinbase + Cloudflare's HTTP 402 micropayment protocol
  • ACP — Agent Commerce Protocol (Stripe + OpenAI)
  • AP2 — Agents to Payments (Google Cloud + 60 partners)
  • Visa TAP — Trusted Agent Protocol

PaySentry acts as a universal middleware layer that works across all payment rails.

How does circuit breaker protection work in PaySentry?

Circuit breakers in PaySentry automatically pause agent spending when anomalies are detected:

  • Rate spike detection — If an agent fires 10 payments in 30 seconds (vs. normal 2-3 per hour), the circuit breaker triggers
  • Statistical anomaly detection — If an agent makes a $499 payment when its historical average is $12, the circuit breaker halts all transactions

When triggered, the circuit breaker pauses all transactions from that agent until manual review. This prevents runaway spending from prompt injection or agent malfunction.

Can I use PaySentry for compliance with the EU AI Act?

Yes. PaySentry generates compliance-ready logs that satisfy:

  • EU AI Act Article 14 — Automatic logging of events throughout the system lifecycle
  • EU AI Act Article 72 — Explainability (you must explain why the AI made a specific decision)

Every transaction includes: intent chain, policy evaluation result, matched rules, and context snapshots. This provides the audit trail required for high-risk AI systems making autonomous financial decisions.

How do I test agent payment policies before production?

Use PaySentry's sandbox mode with pre-built attack scenarios: 

import { SCENARIO_OVERSPEND, SCENARIO_RATE_SPIKE } from '@paysentry/sandbox';

// Test budget enforcement
await scenarios.run(SCENARIO_OVERSPEND, {
  policy: myPolicy,
  expectedOutcomes: ['completed', 'completed', 'rejected'],
});

// Test rate limiting
await scenarios.run(SCENARIO_RATE_SPIKE, {
  policy: myPolicy,
  expectAlerts: ['rate_spike'],
});

You can also build custom scenarios that simulate prompt injection attacks. The sandbox validates defenses without moving real funds.

What agent frameworks does PaySentry integrate with?

PaySentry integrates with all major agent frameworks:

  • OpenClaw
  • LangChain
  • CrewAI
  • AutoGen
  • Any framework that makes programmatic payment calls

Integration is framework-agnostic — simply wrap your payment execution logic with PaySentry's evaluatePayment() middleware: 

const result = await paysentry.evaluatePayment({
  agentId: 'research-agent',
  recipient: 'https://api.openai.com/v1/chat',
  amount: 0.50,
  currency: 'USDC',
  protocol: 'x402',
});

if (result.allowed) {
  await executePayment(result.transaction);
}
